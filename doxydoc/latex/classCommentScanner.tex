Invokes the comment block parser with the request to parse a single comment block. 
\begin{DoxyParams}[1]{Parameters}
\mbox{\texttt{ in}}  & {\em parser} & The language parse that invoked this function. The comment block parse may invoke Parser\+Interface\+::parse\+Prototype() in order to parse the argument of a @fn command. \\
\hline
\mbox{\texttt{ in}}  & {\em cur\+Entry} & The \mbox{\hyperlink{classEntry}{Entry}} to which the comment block belongs. Any information (like documentation) that is found in the comment block will be stored in this entry. \\
\hline
\mbox{\texttt{ in}}  & {\em comment} & A string representing the actual comment block. Note that leading $\ast$\textquotesingle{}s are already stripped from the comment block. \\
\hline
\mbox{\texttt{ in}}  & {\em file\+Name} & The name of the file in which the comment is found. Mainly used for producing warnings. \\
\hline
\mbox{\texttt{ in,out}}  & {\em line\+Nr} & The line number at which the comment block was found. When the function returns it will be set to the last line parsed. \\
\hline
\mbox{\texttt{ in}}  & {\em is\+Brief} & TRUE iff this comment block represents a brief description. \\
\hline
\mbox{\texttt{ in}}  & {\em is\+Javadoc\+Style} & TRUE iff this comment block is in \char`\"{}\+Javadoc\char`\"{} style. This means that it starts as a brief description until the end of the sentences is found and then proceeds as a detailed description. \\
\hline
\mbox{\texttt{ in}}  & {\em is\+Inbody} & TRUE iff this comment block is located in the body of a function. \\
\hline
\mbox{\texttt{ in,out}}  & {\em prot} & The protection level in which this comment block was found. Commands in the comment block may override this. \\
\hline
\mbox{\texttt{ in,out}}  & {\em position} & The character position within {\itshape comment} where the comment block starts. Typically used in case the comment block contains multiple structural commands. \\
\hline
\mbox{\texttt{ out}}  & {\em new\+Entry\+Needed} & Boolean that is TRUE if the comment block parser finds that a the comment block finishes the entry and a new one needs to be started. \\
\hline
\mbox{\texttt{ in}}  & {\em markdown\+Enabled} & Indicates if markdown specific processing should be done. \\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
TRUE if the comment requires further processing. The parameter {\itshape new\+Entry\+Needed} will typically be true in this case and {\itshape position} will indicate the offset inside the {\itshape comment} string where to proceed parsing. FALSE indicates no further processing is needed. 
\end{DoxyReturn}
